## Status
Accepted

## Context
Core domain concepts require concrete representations to allow deterministic
testing and auditability. These representations must not encode business logic.

## Decisions

### LogicalTime
- Immutable value object
- Backed by a monotonically increasing unsigned integer
- No wall-clock or timestamp semantics

### DecisionClass
- String-based identifier
- Format: `<domain>.<action>.<variant>@v<version>`
- Immutable, validated on creation

### ReasonCode
- Finite set of string literals
- Namespaced (e.g. `ATTENTION_SUPERIOR`)
- No free-text reasons allowed

### AssetId
- Uppercase alphanumeric string
- Represents a base or quote asset
- No market semantics

### ExchangeId
- Finite enum-like value object
- Initial value: `BINANCE_SPOT`
- Extensible via documentation only

## Consequences
- Implemented as explicit value objects
- Structural equality
- Stable, lossless serialization
